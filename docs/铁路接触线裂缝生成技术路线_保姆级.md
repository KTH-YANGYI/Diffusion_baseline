# 铁路接触线裂缝生成技术路线（保姆级）

## 1. 文档目标

本文档只讨论一件事：**如何用 diffusion（扩散模型）生成可用于训练的铁路接触线裂缝图像-掩膜对（image-mask pairs）**。

这里**不讲下游分割和检测怎么训练**，只专注于“生成这一段到底怎么做”。

你的当前数据条件是：

- 裂缝正样本：132 张，已裁剪为 `640×640`
- 裂缝像素占整图比例极低，约 `0.17%`
- 中位数约 `0.087%`
- 100% 骨架位置都 `>= 2 px`
- 图片来自不同角度拍摄的视频抽帧
- 另有约 5000 张正常工件背景

这类任务的关键难点是：

1. **裂缝极细，面积占比极低**，直接整图生成很容易失败。
2. **真实裂缝样本太少**，不适合从零训练大 diffusion 模型。
3. **背景非常多**，说明你更适合做“正常背景 + 受控缺陷植入”，而不是做“纯随机生成整张缺陷图”。

因此，本文推荐的核心路线不是“从零训练一个整图生成器”，而是：

> **用真实或轻度扰动后的裂缝 mask，驱动一个预训练 inpainting diffusion 模型，在大量正常背景上局部植入裂缝外观。**

这条路线最适合你目前的数据结构，也最适合 diffusion 新手快速做出第一版可用系统。

---

## 2. 先给最终推荐路线

### 2.1 第一版就用这条主线

建议你的第一版生成系统直接按下面的技术栈来：

- 框架：**Hugging Face Diffusers**
- 主生成模型：`stable-diffusion-v1-5/stable-diffusion-inpainting`
- 背景保持控制模型：`lllyasviel/control_v11p_sd15_inpaint`
- 域适配微调模型：在 `stable-diffusion-v1-5/stable-diffusion-v1-5` 上训练一个 **LoRA**

### 2.2 为什么先用这条路线

因为你当前最适合的是：

- 用 **预训练模型** 解决“裂缝外观生成”
- 用 **mask** 解决“裂缝位置与几何”
- 用 **正常背景库** 解决“背景多样性”

而不是一上来就尝试：

- 从零训练 diffusion
- 直接生成整张裂缝图
- 让 diffusion 同时生成图像和精确 mask

### 2.3 一句话理解整个系统

你的系统本质上不是“画一张图”，而是：

> 给模型一张正常工件图、一张裂缝 mask、一个简短提示词，让它只在 mask 区域里长出一条逼真的裂缝，而且尽量不破坏周围背景。

## 3. 术语先讲清楚

这一节专门给 diffusion 新手看。

### 3.1 checkpoint

**checkpoint** 就是训练好的模型权重。

你可以把它理解成：

- 模型的一个现成版本
- 你下载下来就能直接推理或继续微调的东西

比如：

- `stable-diffusion-v1-5/stable-diffusion-inpainting`
- `lllyasviel/control_v11p_sd15_inpaint`

这些都是现成 checkpoint。

### 3.2 inpainting

**inpainting** 中文可以理解为“局部重绘”或“局部修补生成”。

它的输入通常包括：

- 原图
- mask
- prompt（提示词）

它的含义是：

- **mask 白色区域**：允许模型修改
- **mask 黑色区域**：尽量保持原图不变

这正好非常适合你的任务：

- 原图：正常工件背景
- mask：裂缝位置
- 输出：带裂缝的工件图

### 3.3 mask

**mask** 就是一个二值图。

在你的任务里：

- 裂缝位置用白色表示
- 背景用黑色表示

但是在工程实现里，建议你保留两张 mask：

#### `mask_raw`

- 原始人工标注的精确裂缝 mask
- 用来作为最终 image-mask pair 里的标签

#### `mask_edit`

- 生成时给 diffusion 用的编辑区域 mask
- 一般由 `mask_raw` 轻微膨胀得到
- 作用是让模型“看得见”裂缝该长在哪里

### 3.4 LoRA

**LoRA** 是一种轻量微调方法。

它不是把整个大模型重训，而是：

- 在模型里插入一小部分可训练参数
- 只训练这一小部分

这样有几个好处：

- 显存需求低
- 训练速度快
- 对小数据集更友好
- 训练后得到的附加权重文件很小

在你的项目里，LoRA 的作用不是学习“裂缝具体位置”，而是学习：

- 你的工件表面质感
- 你的拍摄光照风格
- 你的裂缝长相

### 3.5 ControlNet

**ControlNet** 是 diffusion 的一个结构控制模块。

它的作用是：

- 除了原图和提示词以外
- 再额外喂给模型一张“控制图”
- 让模型更听这张图的话

在你的项目里，ControlNet 可以用来：

- 更好地保持背景不被破坏
- 更严格地控制裂缝的形状、走向、宽度

### 3.6 prompt

**prompt** 就是提示词。

你告诉模型：

- 这是工业金属表面
- 这是近景特写
- 裂缝是发丝状还是更严重的裂缝
- 图像是单色、真实、清晰的

### 3.7 negative prompt

**negative prompt** 是反向提示词。

你告诉模型：

- 不要卡通感
- 不要彩色污染
- 不要多条裂缝
- 不要水印
- 不要背景断裂

### 3.8 strength

**strength** 可以理解为“修改强度”。

- 越高：模型越敢改
- 越低：越接近原图

### 3.9 guidance scale

**guidance scale** 可以理解为“模型听 prompt 的程度”。

- 越高：越听提示词
- 越低：越自由

### 3.10 padding_mask_crop

这个参数非常重要。

它的作用是：

- 不直接在整张图上处理
- 而是先把 mask 周围区域裁出来
- 放大后局部做 inpaint
- 再贴回原图

因为你的裂缝很细，所以这个机制对你特别有价值。

### 3.11 apply_overlay

这是一个背景保护技巧。

它的作用是：

- 强制把 mask 外的区域尽量保持为原图
- 只让生成主要发生在 mask 内或边界附近

对工业场景很实用，因为你比起“艺术过渡自然”，更在意“背景不要被改坏”。

## 4. 这条路线的核心建模思路

你的第一版生成系统，建议建模成这样：

\[
I_{syn} = G(B, M_{edit}, P, C)
\]

其中：

- `B`：正常背景 patch
- `M_edit`：编辑区域 mask
- `P`：prompt
- `C`：可选控制图（ControlNet 用）
- `I_syn`：生成出来的带裂缝 patch

也就是说，模型不是凭空画一张图，而是在正常背景上，按 mask 指定的区域去“长”出裂缝。

这条思路适合你，原因有三点：

1. 你有大量正常背景。
2. 你有少量真实裂缝 mask。
3. 你的裂缝很细，不适合做无约束整图生成。

## 5. 第一步：先把数据整理成适合 diffusion 的格式

这一部分非常关键。

如果数据组织方式不对，后面即使模型选对了，也很难生成出稳定结果。

### 5.1 不要直接拿整张 640×640 做第一版生成单元

虽然你的数据已经是 `640×640`，但第一版我不建议直接整图做生成。

原因：

- 裂缝面积太小
- diffusion 在整图里很容易把注意力都放到背景上
- 结果就是背景很真实，裂缝不明显或裂缝不受控

### 5.2 正确做法：先裁裂缝 ROI patch

对每一张真实裂缝图：

1. 取裂缝 mask 的最小外接框
2. 向四周扩边界 `64～128 px`
3. 最后统一缩放或裁成 `512×512`

第一版建议你直接固定：

- **patch 尺寸 = 512×512**

为什么是 512：

- SD1.5 inpainting 对 512 非常成熟
- patch 足够大，既保留裂缝局部结构，也保留一定背景上下文
- 后续如果升级到 SDXL，512 也是安全起点

### 5.3 如果原图是灰度图，怎么喂给模型

大多数公开 Stable Diffusion checkpoint 默认接受 3 通道图像。

如果你的工件图是灰度图，第一版最简单的做法是：

- 直接复制 3 份，变成伪 RGB 图

也就是：

- R 通道 = 灰度图
- G 通道 = 灰度图
- B 通道 = 灰度图

这样虽然不优雅，但非常实用，而且兼容性最好。

### 5.4 每个真实裂缝样本建议准备哪些内容

建议为每个裂缝 patch 准备以下内容：

#### 1）`img_crack`

- 原始裂缝 patch 图像

#### 2）`mask_raw`

- 精确裂缝标签
- 后续 image-mask pair 中的正式 mask

#### 3）`mask_edit`

- 由 `mask_raw` 膨胀得到的编辑区域
- 建议膨胀 `2～4 px`

为什么要膨胀：

- 裂缝太细时，模型容易“看不见”这么窄的编辑区域
- 给它略宽一点的白色带，模型更容易把裂缝纹理画进去

#### 4）`meta_stage`

裂缝阶段标签，哪怕很粗也有用。

比如：

- 早期：很细、发丝状
- 中期：明显延长或略变宽
- 晚期：接近断裂、裂缝开口明显

#### 5）`meta_view`

记录一些基础属性：

- 视频编号
- 相机角度
- 光照条件
- 批次信息

即使第一版不直接喂给模型，这些元数据也有利于后面做更合理的背景匹配和采样。

### 5.5 正常背景库怎么建

你有 5000 张正常背景，这是非常宝贵的数据。

但不要直接把这 5000 张图原封不动当背景池。

建议你也对正常图做同样的 patch 化：

1. 只保留工件区域
2. 只裁那些“现实中裂缝可能出现的区域”
3. 裁成 `512×512`
4. 按角度、亮度、表面纹理做简单分组

为什么要分组：

- 如果 donor mask 来自某种视角
- 你最好从相似视角的正常背景里挑 patch
- 这样生成结果会更自然

## 6. 第二步：第一版先做“不训练”的 baseline

这里的目标不是一下做到最强，而是：

> 先把“正常背景 + mask + inpainting = 裂缝图”这条流程跑通。

### 6.1 第一版建议使用的现成 checkpoint

#### 主模型

`stable-diffusion-v1-5/stable-diffusion-inpainting`

它是第一版最推荐的主模型。

原因：

- 是成熟的 inpainting 模型
- 社区使用广泛
- 工程稳定性好
- 对新手最友好

#### 背景控制模型

`lllyasviel/control_v11p_sd15_inpaint`

它是与 SD1.5 inpaint 体系兼容的 ControlNet。

它的作用不是直接告诉模型“裂缝长什么样”，而是：

- 告诉模型哪里是待编辑区域
- 帮助模型更好保持原图背景
- 减少 mask 外区域被误改的概率

### 6.2 这一版推理时的输入四元组

每次生成都可以看成这样一组输入：

\[
(B, M_{edit}, P, C)
\]

其中：

- `B`：正常背景 patch
- `M_edit`：膨胀后的编辑 mask
- `P`：prompt
- `C`：ControlNet 控制图

### 6.3 第一版的核心动作到底是什么

可以用一句话概括：

> 从正常背景图中，按你指定的细长 mask 区域，局部植入一条真实感裂缝。

### 6.4 第一版的 ControlNet 控制图怎么构造

第一版不建议你自己设计复杂控制图。

直接使用 inpaint ControlNet 的思路：

- 给模型一张“带孔洞的背景图”
- 孔洞的位置由 `mask_edit` 决定

更直白地说：

- 背景里 mask 区域被挖空
- 模型知道这里应该补内容
- 其他区域尽量保持原样

这种方式特别适合第一版流程验证。

### 6.5 第一版的 prompt 应该怎么写

工业生成的 prompt 不要写得太花。

你的目标不是画艺术图，而是让模型稳定地产出“工业金属表面裂缝”。

建议你一开始只固定 3 套模板。

#### 模板 A：早期细裂纹

`close-up industrial metallic contact wire surface, thin hairline crack, realistic, macro photo, monochrome, sharp focus`

解释：

- `close-up`：近距离特写
- `industrial metallic`：工业金属表面
- `contact wire surface`：接触线工件表面
- `thin hairline crack`：很细的发丝状裂纹
- `realistic`：真实风格
- `macro photo`：微距照片感
- `monochrome`：单色/灰度风格
- `sharp focus`：清晰成像

#### 模板 B：中期裂纹

`close-up industrial metallic contact wire surface, growing crack, realistic, macro photo, monochrome, sharp focus`

#### 模板 C：晚期裂纹

`close-up industrial metallic contact wire surface, severe crack near fracture, realistic, macro photo, monochrome, sharp focus`

### 6.6 negative prompt 应该怎么写

建议第一版固定使用一条反向提示词，不要频繁改：

`cartoon, painting, colorful, rust, dirt, text, watermark, blur, low contrast, duplicate crack, double crack, extra branches, broken background`

它的作用是尽量压制这些常见问题：

- 卡通感
- 绘画感
- 彩色污染
- 生锈、脏污等不符合你任务的元素
- 水印
- 模糊
- 背景开裂
- 一次生成出两条裂缝
- 多余分叉

### 6.7 第一版参数建议

下面给你一套适合开局的参数范围。

这些参数不是绝对正确答案，但非常适合作为起点。

#### 1）分辨率

- 固定 `512×512`

#### 2）`num_inference_steps`

- 先用 `30～40`

解释：

- 步数太少，细节可能不够
- 步数太多，时间变长，而且不一定更自然

第一版 30～40 足够。

#### 3）`strength`

建议按裂缝阶段来设：

- 早期细裂纹：`0.75 ~ 0.85`
- 中期裂纹：`0.85 ~ 0.92`
- 晚期裂纹：`0.92 ~ 1.00`

理解方式：

- 细裂纹只需要轻度改动背景
- 更严重的裂纹需要更大胆地改动局部区域

#### 4）`guidance_scale`

建议起始范围：

- `4.5 ~ 7.0`

经验理解：

- 太低：模型不听 prompt，裂缝不明显
- 太高：可能画得太“用力”，出现不自然的硬裂纹

#### 5）`padding_mask_crop`

建议起始：

- 短裂缝：`32`
- 中长裂缝：`48`
- 更长裂缝：`64` 或 `96`

这个参数的意义是：

- 把裂缝附近放大后再画
- 对极细结构非常有帮助

#### 6）`mask blur`

建议：

- 很细裂纹：`0 ~ 2`
- 稍宽裂纹：`2 ~ 4`

你的裂缝目标很细，所以不要把这个值调太大。

如果太大：

- 边缘会糊
- 裂缝会变得像涂抹痕迹

#### 7）`controlnet_conditioning_scale`

建议起始：

- `0.8 ~ 1.1`

理解方式：

- 值更高：ControlNet 更强，背景更稳，但模型可能变“死”
- 值更低：模型自由度更高，但可能更容易破坏背景

### 6.8 `apply_overlay` 的建议

对于你的工业裂缝任务，第一版建议：

- **默认开启**

原因：

- 你更怕背景被改坏
- 工业图像里，背景异常修改会严重污染数据
- 边界过渡略硬，通常比背景乱改更容易接受

### 6.9 一组背景与 donor mask 不要只生成一张

扩散模型天生有随机性。

因此，建议每个组合都这么做：

- 背景 patch 固定
- mask 固定
- prompt 固定
- 参数固定
- 只改变随机种子 `seed`
- 一次生成 `4` 张候选图

然后你再人工挑选或后续过滤。

这样做的好处是：

- 同样一组条件下，自然可以得到多样性
- 不需要不停改 prompt
- 更容易分析哪些图是好结果

### 6.10 第一版 pair 怎么保存

建议每个样本都保存以下元信息：

- `image_syn`
- `mask_raw`
- `mask_edit`
- `bg_id`
- `donor_id`
- `stage`
- `prompt`
- `negative_prompt`
- `seed`
- `strength`
- `guidance_scale`
- `padding_mask_crop`
- `control_scale`
- `checkpoint_name`

为什么这么麻烦：

因为生成系统一旦规模变大，你一定会遇到下面这些问题：

- 某一批图突然不自然
- 某个参数组合总爱生成双裂缝
- 某个 donor mask 总会失败

如果不记录元信息，你后面很难追溯原因。

## 7. 第三步：第一版跑通以后，再加 LoRA 学你的工业域风格

这一阶段的目的不是控制裂缝位置，而是：

> 让裂缝看起来更像你的真实工件表面上长出来的。

### 7.1 为什么要加 LoRA

如果你只用通用的 inpainting checkpoint，通常会出现这些问题：

- 裂缝像是“自然图像里的裂缝”
- 不是很像你真实工件的金属表面
- 局部反光、灰度过渡、表面质感不够贴近真实域

这时 LoRA 就派上用场了。

### 7.2 LoRA 应该训练在哪个 checkpoint 上

建议第一版 LoRA 训练在：

- `stable-diffusion-v1-5/stable-diffusion-v1-5`

而不是直接在 inpainting checkpoint 上训练。

原因：

1. SD1.5 base 的 LoRA 生态最成熟。
2. 你的 LoRA 目标是学习“工业表面风格”和“裂缝外观风格”。
3. 训练好的 LoRA 权重可以再加载回 inpainting 管线中使用。

### 7.3 LoRA 的训练集怎么组织

LoRA 不建议只喂 132 张正样本。

更合理的方案是：

- 裂缝 patch：132 张左右
- 正常 patch：从 5000 张正常背景中采样 1000～2000 张

这样 LoRA 学到的是：

- 这个工件表面本身长什么样
- 裂缝和正常区域之间有什么视觉区别
- 在你的拍摄条件下，灰度和反光是怎么分布的

### 7.4 LoRA 训练用 caption 怎么写

caption 一定要尽量固定，尤其你数据量很少。

#### 正常 patch caption 模板

`close-up industrial metallic contact wire surface, realistic, macro photo, monochrome, sharp focus`

#### 裂缝 patch caption 模板

`close-up industrial metallic contact wire surface with a thin hairline crack, realistic, macro photo, monochrome, sharp focus`

如果你有阶段标签，则替换中间的裂缝词：

- `incipient hairline crack`
- `growing crack`
- `severe crack near fracture`

### 7.5 LoRA 初始超参数建议

建议你先从一组保守参数起步：

- `resolution = 512`
- `rank = 8`
- `learning_rate = 1e-4`
- `batch_size = 1`
- `gradient_accumulation = 4`
- 训练步数：先看 `2000～5000 step`

这些值的作用这样理解：

#### `rank`

控制 LoRA 可学习容量。

- 太小：可能学不进去你的风格
- 太大：更容易过拟合

所以第一版先从 `8` 开始。

如果你发现：

- 学得不明显
- 裂缝风格没有明显变好

再试：

- `rank = 16`

#### `learning_rate`

建议起点：

- `1e-4`

LoRA 通常可以用比全量微调更高的学习率。

#### `batch_size = 1`

这是很常见的起点，尤其是显存有限时。

#### `gradient_accumulation = 4`

如果显存不够大，用梯度累积是很正常的做法。

### 7.6 LoRA 训练过程中你该看什么

不要只盯着 loss。

你应该固定几条验证 prompt，隔一段时间就生成几张图看效果。

你要重点看：

1. 工件表面是不是越来越像你的真实数据
2. 裂缝是不是更自然
3. 是否开始“到处都想长裂缝”
4. 是否开始记忆训练图背景

如果出现下面现象，通常说明过拟合了：

- 表面纹理越来越像某几张训练图
- 无裂缝 prompt 也容易长裂缝
- 裂缝外观开始模式化、模板化

### 7.7 LoRA 训练完以后怎么接回生成系统

接回方式是：

- 主模型仍然是 `stable-diffusion-v1-5/stable-diffusion-inpainting`
- 背景控制仍然是 `lllyasviel/control_v11p_sd15_inpaint`
- 再额外加载你训练好的 LoRA

这样三者分工就很清楚：

- **inpainting**：负责局部重绘
- **ControlNet**：负责保背景和空间一致性
- **LoRA**：负责学你的工业域风格

## 8. 第四步：如果几何控制还是不够稳，再做自定义 ControlNet

这一阶段不是第一版必做项。

只有当你发现下面问题持续存在时，才建议做：

- 裂缝总是偏离你给定的 mask 中线
- 裂缝宽度变化不对
- 裂缝容易莫名分叉
- 裂缝明明画出来了，但拓扑不稳定

### 8.1 为什么第一版不建议立刻做自定义 ControlNet

因为它比直接用现成 inpaint ControlNet 更复杂：

- 需要你自己设计控制图
- 需要你训练额外模型
- 需要更多调参

对 diffusion 新手来说，正确顺序应该是：

1. 先用现成 checkpoint 跑通
2. 再用 LoRA 学风格
3. 最后再考虑做更精细的几何控制

### 8.2 自定义 ControlNet 在这里要解决什么问题

它的目标不是让裂缝“更真实”，而是让裂缝：

- 更听你的几何条件
- 更贴着骨架走
- 更符合你给定的局部宽度变化

### 8.3 你的自定义控制图建议怎么设计

建议做成 **3 通道控制图**：

#### 通道 1：`mask_edit`

告诉模型：

- 哪些区域允许画裂缝

#### 通道 2：`skeleton`

也就是裂缝骨架图。

作用是：

- 告诉模型裂缝中线在哪里
- 让模型更清楚裂缝应该沿哪条路径延展

#### 通道 3：`distance transform`

也就是距离变换图。

作用是：

- 告诉模型局部宽度信息
- 哪些地方该更细
- 哪些地方可以稍微宽一点

### 8.4 为什么骨架图和距离变换对你很重要

你的裂缝有两个特点：

1. 很细
2. 形状是细长连通结构

如果只给二值 mask，模型只知道“大概在这片白色区域里画东西”。

但如果再加：

- 骨架图：告诉它裂缝中心线
- 距离变换：告诉它宽度渐变

那么几何控制会更强。

### 8.5 自定义 ControlNet 的训练数据怎么构造

每个训练样本包括：

- 输入控制图：`[mask_edit, skeleton, distance_transform]`
- 原图：正常背景 patch 或孔洞图
- 目标图：真实裂缝 patch
- prompt：固定工业 prompt

它学的是：

> 当我看到这样的几何控制图时，应该长出这样的裂缝外观。

### 8.6 为什么这一阶段仍建议留在 SD1.5 体系里做

因为：

- 工程成熟
- 社区教程多
- 显存要求比 SDXL 体系低
- 新手更容易排查问题

所以顺序上，建议先把 SD1.5 体系做好，再考虑更重的模型。

## 9. 你现在要不要让 diffusion 直接生成 mask

**不建议第一版这样做。**

### 9.1 第一版 mask 应该怎么来

第一版最稳妥的做法是：

- 用真实人工标注的 `mask_raw`
- 再做轻微几何扰动，得到新的 `mask_edit`

这样你可以很快得到大量 image-mask pairs。

### 9.2 mask 轻扰动建议做哪些操作

可以做这些小幅变化：

- 小角度旋转
- 沿主方向轻微弯曲
- 长度轻微缩放
- 局部宽度缩放
- 尖端形状变化

但是不要做这些：

- 无规律的大幅扭曲
- 裂缝拓扑断开
- 无依据地增加大分叉
- 把裂缝改得和真实物理过程完全不符

### 9.3 为什么先不做 mask diffusion

因为你当前正样本只有 132 张。

如果一上来让 diffusion 去学习“裂缝几何的分布”，通常不划算。

更合理的任务分工是：

- **mask 提供几何与位置**
- **diffusion 提供真实外观**

这是当前阶段成功率最高的做法。

### 9.4 什么时候再考虑单独做 mask 生成

当你把 image 生成系统已经跑稳以后，如果发现真正的瓶颈是：

- 可用裂缝几何变化不够多
- donor mask 的形状多样性不足

那时再单独做一个“小型二值 mask 生成器”更合理。

但这已经属于第二阶段升级项，不是第一版必须做的内容。

## 10. 参数调节时的常见现象与应对方法

这一节很实用，主要讲如果结果不好，看哪个参数。

### 10.1 裂缝太弱、几乎看不见

优先检查：

- `strength` 是否太低
- `mask_edit` 是否太窄
- `guidance_scale` 是否太低
- prompt 里的裂缝词是否不够明确

优先处理顺序建议：

1. 先把 `mask_edit` 膨胀一点
2. 再把 `strength` 提高一点
3. 再把 `guidance_scale` 提高一点

### 10.2 背景被改坏了

优先检查：

- `apply_overlay` 是否没有开
- `controlnet_conditioning_scale` 是否太低
- `strength` 是否太高
- `mask_edit` 是否膨胀过头

优先处理顺序：

1. 开启 `apply_overlay`
2. 增大 `controlnet_conditioning_scale`
3. 降低 `strength`
4. 缩小 `mask_edit`

### 10.3 裂缝像笔画，不像真实裂纹

优先检查：

- prompt 是否写得太简单或太泛
- negative prompt 是否没有压掉“cartoon / painting”
- LoRA 是否还没训练
- `guidance_scale` 是否太高导致模型“硬画”

### 10.4 一次总爱画出两条裂缝

优先检查：

- negative prompt 中是否加入 `duplicate crack, double crack`
- `mask_edit` 是否过宽
- `strength` 是否太高
- donor mask 是否本身就形状不稳定

### 10.5 裂缝偏离骨架、跑歪

如果是第一版现成 ControlNet：

- 先缩小编辑区域
- 先提高 `controlnet_conditioning_scale`
- 再尝试降低 `strength`

如果还是不稳：

- 就说明你已经接近需要自定义 ControlNet 的阶段了

### 10.6 边缘过渡太硬

优先检查：

- `apply_overlay` 是否导致边界太生硬
- `mask blur` 是否过小

可以小幅尝试：

- 把 `mask blur` 从 0 调到 1 或 2

但不要太大，否则细裂缝会糊掉。

## 11. 推荐的整体执行顺序

下面是一个最适合你当前阶段的落地顺序。

### 阶段 A：先跑通流程

1. 把真实裂缝图裁成 `512×512` patch
2. 为每个裂缝 patch 准备 `mask_raw` 和 `mask_edit`
3. 把正常背景也裁成 `512×512` patch 库
4. 先用 `stable-diffusion-v1-5/stable-diffusion-inpainting`
5. 加上 `lllyasviel/control_v11p_sd15_inpaint`
6. 固定 prompt 模板和 negative prompt 模板
7. 用多 seed 生成候选图
8. 保存所有生成元信息

### 阶段 B：把图生得更像你的真实域

1. 在 `stable-diffusion-v1-5/stable-diffusion-v1-5` 上训练 LoRA
2. 训练集用“裂缝 patch + 正常 patch”
3. caption 尽量固定、简洁
4. 训练后把 LoRA 挂回 inpainting 管线
5. 再重复阶段 A 的生成流程

### 阶段 C：当几何控制成为瓶颈时再升级

1. 设计三通道控制图：`mask_edit + skeleton + distance transform`
2. 训练自定义 ControlNet
3. 再用于更精细的裂缝几何控制

### 阶段 D：当系统已经稳定，只差画质上限时再升级到 SDXL

1. 保持整个数据流和流程不变
2. 只把主 inpainting 模型升级为 SDXL inpaint
3. 必要时再引入 SDXL refiner

## 13. 你现在最不建议走的路线

最后再强调一下，你当前阶段最不建议做的是下面几件事。

### 13.1 不建议从零训练整图 diffusion

原因：

- 正样本太少
- 裂缝极细
- 容易只学会背景，学不会裂缝

### 13.2 不建议第一版就让 diffusion 同时生成图和 mask

原因：

- mask 的几何精度要求很高
- 图像与 mask 的像素级对齐很难保证
- 你当前样本量不适合直接做这件事

### 13.3 不建议一开始就上 SDXL 全家桶

原因：

- 显存、调参、工程复杂度都更高
- 对新手不友好
- 第一版真正需要解决的是“流程跑通”和“控制稳定”
- 不是“画质极限”

## 14. 最终结论

对你这个铁路接触线裂缝生成任务，最合理、最稳妥、最适合新手的 diffusion 技术路线是：

> **以真实或轻度扰动的裂缝 mask 作为几何先验，使用预训练 inpainting diffusion 模型，在大量正常工件背景上进行局部裂缝植入；先用 SD1.5 inpainting 跑通，再用 LoRA 学工业域风格，如果几何不稳再上自定义 ControlNet。**

如果只保留一句最实用的话，那就是：

> **先不要让 diffusion 自己“发明裂缝位置”，先让它只负责把裂缝“画得真实”。**

这才是最符合你当前数据条件的路线。
