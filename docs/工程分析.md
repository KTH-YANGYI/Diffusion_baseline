# diffusion_v2 架构说明

当前仓库只保留 `diffusion_v2` 一条方法线。源码、配置和产物对应关系如下：

```text
configs/methods/diffusion_v2/contact_wire_v2.yaml
src/crack_synth/methods/diffusion_v2/
artifacts/dataset_real/methods/diffusion_v2/
```

## 模块职责

```text
paired_dataset.py      读取 crack/normal 配对
roi.py                 图片、LabelMe mask、裁剪、膨胀、模糊
prepare_rois.py        生成 roi_assets
guide.py               生成 mask_dark 或 paired_residual 引导图
prepare_lora_data.py   从 ROI 导出 LoRA 训练样本
train_inpaint_lora.py  训练 Stable Diffusion inpainting UNet LoRA
generate.py            加载 inpainting pipeline，执行 v2 生成
postprocess.py         soft overlay、diff 图、启发式质量指标
evaluate_generation.py 生成后离线评估
```

## v2 生成逻辑

`diffusion_v2` 不是直接整图随机生成。它的核心是：

```text
paired normal ROI
+ mask_edit_roi
+ paired residual guide
+ LoRA
+ prompt
-> Stable Diffusion Inpainting
-> soft paste back to 512 ROI
```

当前图片本身已经是 `512x512`，所以第一层 ROI 等价于整图标准化。真正缩小编辑区域的是 `local_inpaint`：

```text
512 background_roi
    -> crop 128/160 around mask_raw
    -> resize crop to 512 for SD inpainting
    -> resize generated crop back
    -> paste into original 512 background
```

## paired residual guide

`crack_prior_mode: paired_residual` 会读取：

```text
background_roi.png
defect_roi.png
mask_raw_roi.png
```

然后计算：

```text
residual = max(background_gray - defect_gray, 0)
```

这个 residual 只在 mask 附近按 `crack_prior_alpha` 叠到 normal ROI 上，作为弱空间引导。它让模型知道裂缝应该沿哪个形状出现，而不是只靠 prompt 猜。

## LoRA 数据流

```text
roi_assets/roi_pairs.jsonl
    -> prepare_lora_data
    -> lora_data/train/<sample_id>/
         condition.png
         background.png
         target.png
         mask_label.png
         mask_train.png
         caption.txt
    -> train_inpaint_lora
    -> lora_ctwirecrack_v2/
```

`condition.png` 是 normal/residual-guided 条件图，`target.png` 是真实 crack crop。训练时只训练 UNet LoRA，VAE 和 text encoder 冻结。

## 为什么清掉旧版本

旧结构里同时存在 v1、guided-inpaint、lora-inpaint、residual-v2，配置输出路径和 CLI 名称互相混杂。现在只保留 v2 后，默认入口、默认配置和默认产物目录都唯一：

```text
prepare_rois
prepare_lora_data
train_inpaint_lora
generate_cracks
evaluate_generation
```
